Event Image Path Helper

This document provides guidance on how to correctly reference images in event pages.

Path Structure

When referencing images in event detail pages, remember to use the parent directory notation ../ because event pages are in a subdirectory.

Important: Use the proper relative paths for all resources when working in event detail pages.

Correct Image Path Examples


<img src="../img/events/event-name.jpg" alt="Event Name">


<img src="../img/acumen-logo.png" alt="ACUMEN Logo">


background-image: url('../img/banners/events_banner.jpg');

Common Path Mistakes


<img src="img/events/event-name.jpg" alt="Event Name">


<img src="../img/events/event-name.jpg" alt="Event Name">

Automatic Path Fixing

Our site includes the path-fixer.js script which attempts to fix incorrect paths, but it's better to use correct paths from the start.

If images are not displaying properly:

  1. Make sure the image actually exists in the referenced location
  2. Check that you're using ../ at the beginning of paths
  3. Verify that the image filename and extension match exactly (case-sensitive)
